# Change Checklist

Use before proposing or implementing significant changes.

## Pre-Change
- [ ] Change clearly scoped and documented
- [ ] Stakeholders identified and informed
- [ ] Rollback plan defined
- [ ] Dependencies mapped

## Implementation
- [ ] Change tested in isolation before integration
- [ ] Existing tests still pass
- [ ] Breaking changes documented

## Post-Change
- [ ] Documentation updated
- [ ] Team notified
- [ ] Monitoring confirmed working
