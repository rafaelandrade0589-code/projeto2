# Component Quality Checklist

Use by @ux-design-expert after `*build` or `*compose`.

## Structure
- [ ] Component follows Atomic Design level (atom/molecule/organism)
- [ ] Props interface documented
- [ ] Default props set where appropriate
- [ ] Component name follows PascalCase convention

## Accessibility
- [ ] ARIA attributes present where needed
- [ ] Keyboard navigable
- [ ] Focus visible state styled
- [ ] Color contrast WCAG AA minimum

## Design Tokens
- [ ] Colors use design tokens (not hardcoded hex)
- [ ] Spacing uses design scale
- [ ] Typography uses defined type scale

## Testing
- [ ] Renders without errors
- [ ] Variants/states tested visually
