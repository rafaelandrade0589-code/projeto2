# Architect Checklist

Use before finalizing architecture decisions.

## Design Decisions
- [ ] Technology selection justified with trade-offs documented
- [ ] Scalability requirements addressed
- [ ] Security boundaries defined
- [ ] Integration patterns chosen and documented
- [ ] Data architecture reviewed

## Documentation
- [ ] Architecture diagram updated
- [ ] ADR (Architecture Decision Record) created if significant change
- [ ] Impact on existing systems assessed

## Review
- [ ] No over-engineering — complexity matches requirements
- [ ] Team can implement within current skill set
- [ ] Monitoring and observability considered
