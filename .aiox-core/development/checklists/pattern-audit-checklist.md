# Pattern Audit Checklist

Use by @ux-design-expert during `*audit` command.

## Discovery
- [ ] All UI components catalogued
- [ ] Similar components grouped (clustering)
- [ ] Redundancy count per component type documented
- [ ] Estimated LOC/tokens duplicated calculated

## Analysis
- [ ] Root cause of redundancy identified (no design system, multiple devs, etc.)
- [ ] Most-used patterns identified as consolidation candidates
- [ ] Risk level assessed for each consolidation

## Output
- [ ] Shock report generated (`*shock-report`)
- [ ] ROI estimate included
- [ ] Recommended consolidation order (highest-impact first)
