# Migration Readiness Checklist

Use by @ux-design-expert before executing design system migration.

## Pre-Migration
- [ ] Component inventory complete
- [ ] Design tokens defined and exported
- [ ] Target component library selected
- [ ] Migration phases documented (4-phase approach)

## Phase 1 Readiness (Foundation)
- [ ] Design tokens integrated
- [ ] Base styles migrated
- [ ] Team trained on new system

## Phase 2 Readiness (Core Components)
- [ ] Atoms replaced (buttons, inputs, typography)
- [ ] No regressions in critical flows

## Phase 3 Readiness (Molecules/Organisms)
- [ ] Complex components migrated
- [ ] Legacy components deprecated (not deleted)

## Phase 4 Readiness (Cleanup)
- [ ] Legacy code removed
- [ ] Bundle size reduced vs baseline
- [ ] Documentation complete
